# [Gold V] 1038: 감소하는 수
[문제 링크](https://www.acmicpc.net/problem/1038)  

### 알고리즘 분류 :  
브루트포스 알고리즘, 백트래킹

### 시간 제한 :  
1 초   

### 문제 설명 :  
음이 아닌 정수 X의 자릿수가 가장 큰 자릿수부터 작은 자릿수까지 감소한다면, 그 수를 감소하는 수라고 한다. 예를 들어, 321과 950은 감소하는 수지만, 322와 958은 아니다. N번째 감소하는 수를 출력하는 프로그램을 작성하시오. 0은 0번째 감소하는 수이고, 1은 1번째 감소하는 수이다. 만약 N번째 감소하는 수가 없다면 -1을 출력한다.  


### 입력 :   
첫째 줄에 N이 주어진다. N은 1,000,000보다 작거나 같은 자연수 또는 0이다.  

### 출력 :   
첫째 줄에 N번째 감소하는 수를 출력한다.  

-----------------------------------------------------------  
### 나의 해석 :  
처음에 예제 입력 50000에서 -1이 나오는 것을 보고 경우의 수가 많지 않을 거라 생각하여  
1부터 10억까지 반복문을 돌리는 것으로 단순하게 생각했었다.  
앞에서부터 감소하는 숫자라는 것은 반대로 말하면 뒤에서 순증가이기 때문에  
현재 num의 마지막 자리가 이전 마지막 자리보다 크다면 함수를 계속 진행하는 형식으로 접근했다.  
하지만 문제를 잠깐 보면 알 수 있듯 문제에서 요구하는 가장 큰 감소하는 수는 9876543210이고,  
이는 int의 범위를 벗어나기 때문에 long long을 사용해야 하며 단순 반복으로는 시간 내에 해결할 수 없다.  
사고를 통해 9다음에는 9가 올 수 없고, 8다음에는 8이 올 수 없는 만큼 경우의 수가 기하급수적으로 줄어듦을 확인할 수 있었다.  
이는 0~9까지의 수를 k개 뽑아 순서대로 나열한 것과 같다는 것을 알게 되었다. ex) 9 7 5 4 3 으로 만들 수 있는 수는 97543밖에 없다.  
결국 숫자를 고르냐/고르지 않냐로 경우의 수가 나뉘기 때문에  
2<sup>10</sup>의 경우의 수가 나오지만 아무것도 선택하지 않을 경우 한가지를 제외하면  
만들 수 있는 경우의 수는 최대 1023가지가 나온다. 
백트래킹을 통해 0~9까지 반복문을 통해 뒤에 한자리씩 붙여가며 마지막 자리보다 작은 수까지만 반복문을 진행하면 된다.  
어차피 위에서 언급했듯 9876543210이 가장 큰 감소하는 수이므로 그 뒤로는 조건문에 의해 수가 더 이상 만들어지지 않는다.  
숫자가 42,421,420, 이런 순서로 만들어지기 때문에 다 만들어진 수들을 정렬하여 그 중 N번째 감소하는 수를 출력하면 된다.

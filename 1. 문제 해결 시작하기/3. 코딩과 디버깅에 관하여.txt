대회에서 중요한 것은 빨리 코드를 작성하는 것보다 읽기 쉬운 코드를 작성하는 것이다.
복잡한 코드는 디버깅이 어렵고, 한번에 정확하게 작성하기 어렵기 때문이다.

좋은 코드를 짜기 위한 원칙
1. 간결한 코드 작성하기 : ex.전역변수 사용(프로그래밍 대회 한정, 실무에서는 사용 어려움), 매크로 사용하기
2. 코드 재사용하기 : 코드를 모듈화하여 함수나 클래스로 분리하여 재사용하기
3. 표준 라이브러리 이용하기 : 이미 수많은 검증을 통해 메모리 관리나 정당성 증명 없이 편하게 사용 가능하고, 팀원이 이해하기도 편하다.
4. 항상 같은 형태로 프로그램 작성하기
5. 일관적이고 명료한 명명법 사용하기 : 모호하지 않은 변수명과 함수명 사용하기
6. 모든 자료 정규화하여 저장하기 : 기약 분수와 같이 정규 표현식 이용하기
7. 코드와 데이터를 분리하기

자주하는 실수
1. 산술 오버플로 : 너무 큰 값, 너무 큰 중간 값에 대해 조심하기
2. 배열 범위 밖 원소 접근
3. 일관되지 않은 범위 표현 : 이는 반 열린 구간 [a,b)을 통해 해결
4. off-by-one 오류 : 범위 판정에서 1 오차 여부 확인
5. 컴파일러 상수 오타 : 오타 조심하기
6. 스택 오버플로
7. 다차원 배열 인덱스 바꿔쓰기 : i,j 순서 헷갈리지 말기
8. 잘못된 비교함수 작성 : 복잡한 비교 함수 작성 시 조심하기
9. 최소, 최대 예외
10. 연산자 우선순위
11. 너무 느린 입출력 방식 선택 : cin, scanf의 입출력 속도 차이
12. 변수 초기화 문제
